网卡操作
ifcnofig 查看网卡信息；
iwconfig 查看无线网卡信息；
ifconfig interface options
启动/关闭网卡接口
ifconfig ethx up/down  启动/关闭某网卡接口
设定网卡接口参数mtu、netmask、broadcast
ifconfig ethx mtu 1500 设定ethx的mtu值
ifconfig ethx xxxxx netmask xxxx broadcast xxxx 设定网卡接口地址、子网、广播地址
ifconfig ethx:x  xxxx  在当前网卡上再虚拟另一个IP地址
ifup/ifdown 是脚本文件，只用/etc/sysconfig/network-scripts下存在ethx才能使用该脚本启动/关闭 ifup ethx。
ifconfig ethx xxx     ifconfig编辑后的ethx只能使用ifconfig ethx down 关闭
/etc/init.d/network start 对整个网卡信息进行操作

修改本机路由信息
route -n 显示本机路由信息，-n参数表示使用ip、端口方式显示结果；
route add/del -net/-host netmask gw/dev
-net/-host xxxx 设置本机连接到xxxx这个网段或者主机的路由
route add -net 192.168.1.0/24 gw aaaa 本机连接到xxxx网域/主机需要经由aaaa路由器转发；
route add -net 192.1681.1.0/24 dev ethx 本机通过ethx直连192.168.1.0/24这个网域；
route add/del default gw xxxx 增加/删除本机默认路由，默认路由只能有一个
route add -net aaaa netmask bbbb dev ethx 设定本机连接到aaaa/bbbb是通过ethx网卡接口联机出去


ip 查看/修改网卡osi第2/3层信息和网卡路由信息：

ip -s link show 显示网卡第2层信息，-s 显示统计数据 
ip link set ethx address/name/mtu/up/down 设定网卡ethx的mac地址、网卡名称、mtu、网卡启动/关闭这些osi第2层面上的参数，用于对网卡osi
第2层参数的设定；

ip -s address show ethx 显示网卡ethx的ip信息，作用于osi第3层；
ip address add/del ip参数 dev ethx label ，如：
ip address add 192.168.1.100/24 broadcast + dev eth0 设定eth0的ip地址信息，broadcast + 表示让系统自动计算广播地址，
ip address add 192.168.1.111/24 broadcast + dev ethx label eth0:0 ,在ethx上新增一个名称是eth0:0、地址是192.168.1.111/24、广播地址
自动计算的虚拟网卡，
ip address del 192.168.1.111/24 dev ethx ，删除在ethx上设定的虚拟网卡信息，
ip address 作用于osi的第3层；

ip route show 显示路由信息；
ip route add/del ip或网段 via xxx(gateway) dev ethx  ，当到目的主机/网段与本机直接相连，"via gateway"可以没有，但dev ethx是必须有的。 
ip route add 192.168.5.0/24 dev ethx   192.168.1.0/24网段的主机通过网卡ethx与本机直接相连； 
ip route add 192.168.5.0/24 via 10.10.10.1 dev ethx 主机与192.168.1.0/24网段主机通信需要经过ethx接口将信息发送到网关10.10.10.1，由
网关转发；
ip route del 192.168.5.0/24  删除主机到达192.168.5.0/24网段的路由配置信息；

ping -b 192.168.5.255(broadcast) 对整个网段主机进行ping动作，探测存活主机；
-c 5 执行次数，-n 不进行ip与主机名称的反查；

traceroute -n IP/域名 追踪2台主机间通讯通过的的各个节点的网络状况，-n 不进行主机名称的解析直接使用ip进行探测；

netstat -rn 查看系统路由信息，同route -n
netstat -tunlp 查看本机目前正在监听的网络服务信息，-t/u 仅列出tcp/udp封包的联机，-l 仅列出正在监听的服务，-n 使用ip与端口号而不是
主机名称与服务名称进行显示，-p 列出pid与进程名称。
客户端会随机启动一个大于1024的端口发起与服务器的联机请求。
kill -15 sshd 杀死sshd服务；
/etc/services 服务名称与端口号的对应文档。

远程联机指令：
telnet host/ip port  连接远程主机的某个服务，端口号对应服务；
telnet localhost 22 查看本机的22端口是否正常启动；

查询主机名称与ip地址的对应：
host www.baidu.com 使用/etc/resolve.conf文件中设定的dns服务器，查询www.baidu.com对应的ip地址；
host www.baidu.com 202.134.13.25 使用202.134.13.25这台dns服务器查询www.baidu.com对应的ip地址；

ftp host/ip port ftp到远程主机的指定端口，默认时通过21端口连接；
连接到远程主机后的操作指令：
cd 变换所在的远程主机目录，
dir 列出所在远程主机当前目录内容，
get filename 下载filename这个文件，
mget filename 下载filename这个文件或目录，
put filename  上传filename这个文件到远程ftp服务器上，
delete filename  删除远程主机的filename这个文件，
mkdir dir  在远程主机上新建dir这个目录
lcd /tmp/yaya 切换到本地端主机的/tmp/yaya目录，
passive 启动/关闭passive模式，
binary 数据传输模式设定为binary格式，
help 显示可用命令；
lftp -p port -u user，password host/ip 使用lftp连接ftp服务器。


在文字接口模式下查看xx.html文件或浏览html网页：
lynx xx.html 查看html格式的文件xx.html；
lynx http://www.kernel.org 查看网页；
使用指定编码查看xx.html文件：
lynx -assume_charset=big5 xx.html 指定使用big5编码查看xx.html文件；
LANG=zh_TW.big5 ；lynx xx.html 使用big5编码查看xx.html文件；
/etc/lynx.cfg lynx查看文件时预设使用的编码设定文件，修改参数CHARACTER_SET、ASSUME_CHARSET、PREFERRED_LANGUAGE

在文字接口模式下下载网页数据：
wget http://kernel.org/11/22.txt  
/etc/wgetrc 修改wget下载网页数据时使用的proxy信息，http_proxy、ftp_proxy、use_proxy 3个参数；

封包数据抓取：
tcpdump -i ethx tcp port 21 and host 192.168.1.100 -nn -w 11.txt
-i 指定监听的网口；
-nn 使用ip、portnumber而不是主机名称、服务名称进行显示；
-w filename 将抓取的封包保存成文件filename；
-r filename 读取tcpdump抓取的封包；
-c 10000 指定抓取的封包个数；
host 192.168.1.100 针对单个主机进行封包抓取；
net 192.168 针对网段进行封包抓取；
src host 192.168.1.10 dst net 10.10 抓取来源主机是192.168.1.10目的网段是10.10的封包；
tcp/udp/ether port 21 针对21端口的指定通讯协议抓取封包；

nc host/ip  port  连接到某一端口查看对应服务是否正常启动，
nc localhost 22  查看本机sshd服务是否正常；
nc -l -p 20000  开启本机的20000端口并监听，等待客户端联机；
