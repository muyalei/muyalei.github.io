sector扇区是磁盘的最小物理存储单元，一般为512b
block逻辑区块是磁盘的最小数据存储单元，其中存储相关文件或目录数据，文件或目录属性信息存储在inode中

磁盘分割:向操作系统指定磁盘partition分区的起始磁柱与结束磁柱。
格式化：将每一块partition规划成inode和block两部分，每一块partition的第一个block称为superblock，记录inode与block的信息。

/lib/modules/`uname -r`/fs 记录系统支持的filesystem种类

df -aT，查看系统全部partition信息，包括内存，-T 同时显示各分区的filesystem类型
df -h，以易读方式显示磁盘分区信息
df -i，显示磁盘各分区inode信息，已用、未使用等信息


制作连结文件
ln  sourcefile  destinationfile 
-s，制作symbolic link文件。不加该参数是hard link 连结文件
-f，强制建立连结档
hard link是将被link文件/etc/11.txt的metadata复制一份到/tmp/11.txt，删除/etc/11.txt不影响/tmp/11.txt文件的读取；
symbolic link 是新建一个指向文件，指向被link的文件/etc/11.txt，删除/etc/11.txt后连结档会打不开。

磁盘分割fdisk：
hda、hdb分别指第1、2块物理硬盘，hda1、hda2指hda硬盘的1、2partition；
fdisk针对整个物理磁盘使用；
fdisk /dev/sda，p查看磁盘分区状态，d删除指定分区，n增加一个分区，w将改动写入磁盘分割表，q不储存操作退出；
fdisk -l 列出当前系统所有装置的partition(包括usb设备)；

磁盘格式化
mke2fs -b 4096 /dev/hda3
-b 设定block大小，默认4096b，即8个sector
-i 设定多少block分配一个inode
-c 快速读取检测；-c -c 进行读写检测，会很慢
-j 格式化为ext3格式，ext3是日志式(journal)档案系统
-L 设定表头label名称
mke2fs默认用ext2格式化

mkbootdist --device /dev/fd0 `uname -r` ，制作开机启动软碟

磁盘检测
fsck -C -t ext3 /dev/hd3 
-t 指定filesystem类型
-C 显示检测进度
-A 根据/etc/fstab内容对整个磁盘进行检测，与开机时磁盘检测相同
-a 自动修复坏道，某些filesystem是-y
-f 强制检测
使用fsck进行磁盘检测时磁盘需卸载

挂载、卸载
mount -t ext2 /dev/hda3 /home 将/dev/hda3挂载到/home目录下
-a 根据/etc/fstab内容，将全部partition挂载
-n 略过挂载时将挂载点信息记录到/etc/mtab的步骤
-L 使用表头名称label挂载磁盘分区
-t 指定挂载partition时使用的filesystem类型
-o 指定额外参数。mount -o defaults /dev/sda3 /home ，以默认的额外参数设定将sda3挂载到/home下；mount -o remount /dev/sda3 重新挂载sda3
整个系统filesystem与mount point的对应关系在/etc/fstab
unmount /dev/sda3 

将usb随身盘挂载到linux系统上
使用dmesg命令查看驱动是否捉到u盘
使用fdisk -l 名令查看是否载入u盘成功
mount /dev/sdb1 /mnt/usb ，将U盘挂载到/mnt/usb目录下（没有该目录就新建一个）

/etc/fstab，开机挂载磁盘partition的预设定文件，/etc/mtab、/proc/mounts ，系统实际挂载磁盘partition的记录文件
/etc/fstab数据错误会导致无法正常开机而进入单人维护模式，此时磁盘是read only模式，使用mount -n -o remount，rw /  ，不更新/etc/mtab
文件，同时将 / 重新挂载成可读写模式，修正错误后可正常开机。

建立虚拟内存
将某一分区设定成为swap，存放在内存中运行的程序数据:
fdisk /dev/hda ,磁盘划分时划分出一块partition如hda2作为swap使用，将该partition的ID代号改为82；
mkswap /dev/hda2，将hda2格式化成swap的档案格式；
swapon /dev/hda2，启动虚拟内存/dev/hda2；
swapoff /dev/hda2
在现有文件系统上规划出一块磁盘空间作为 swap file：
if=/dev/zero of=/tmp/swap bs=4k count=16384 ，在/tmp下建立64M的档案装置；
mkswap /tmp/swap ，将该档案格式化成swap档案格式；
swapon /tmp/swap，启动该swap file 
swapoff /tmp/swap
